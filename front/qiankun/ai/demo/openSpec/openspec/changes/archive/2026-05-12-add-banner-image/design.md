## Context

当前首页 Hero 区为纯文字驱动布局：主标题（h1）→ 副标题（p）→ 双 CTA 按钮，垂直居中于全屏。现在需要在副标题与 CTA 之间插入一张 banner 图片，在保持极简调性的前提下增加视觉锚点。

现有文件：`index.html`（44 行）、`style.css`（194 行），纯 HTML/CSS 无 JS。

## Goals / Non-Goals

**Goals:**
- 在 Hero 区插入 banner 图片，增强首屏视觉冲击力
- 图片响应式适配桌面端和移动端
- 保持整体极简调性，图片不喧宾夺主
- 优雅处理图片加载状态

**Non-Goals:**
- 不做多图轮播
- 不做图片懒加载（纯 HTML 无 JS）
- 不做 WebP/AVIF 格式适配
- 不做图片 CDN 或优化

## Decisions

### 1. 图片位置：副标题与 CTA 之间

**选择**: 在 `hero__subtitle` 和 `cta-group` 之间插入 `<img>`

**理由**: 用户先看到标题理解定位，再看产品图加深印象，最后 CTA 引导行动。这是最自然的阅读节奏：文字 → 视觉 → 行动。

**备选方案**:
- 标题上方：图片先于文字，削弱标题的第一印象
- 页面背景：作为背景图，文字叠在图上，破坏白底黑字极简调性

### 2. 图片尺寸策略：宽度受限 + 自然比例

**选择**: 图片最大宽度 600px（桌面端）/ 100%（移动端），保持原始宽高比

**理由**: 限制最大宽度防止图片占满屏幕喧宾夺主，保持原始比例避免变形。移动端铺满宽度提供最佳视觉体验。

### 3. 图片资源格式与位置

**选择**: 图片存放为 `assets/banner.jpg`，使用 `<img>` 标签直接引用

**理由**: 项目无构建工具，使用相对路径引用最简单。JPEG 适合照片类 banner，文件体积优于 PNG。新建 `assets/` 目录存放静态资源，为后续扩展预留结构。

**备选方案**:
- 根目录放图片：扁平但混乱
- 外部 URL 引用：引入外部依赖，不可控

### 4. 加载占位：纯 CSS 背景色

**选择**: 为 `<img>` 设置 `background-color: #f5f5f5` 作为加载占位

**理由**: 零 JS 依赖下最简单的占位方案。浅灰色暗示"内容即将出现"，不干扰白底黑字调性。

### 5. 间距调整

**选择**: banner 图片与副标题间距 32px（margin-top），与 CTA 按钮间距 32px（margin-bottom）

**理由**: 原有副标题到 CTA 间距为 40px（margin-top on cta-group）。插入图片后，将 cta-group 的 margin-top 保持 40px，图片上方 margin-top 32px，形成 副标题(16px)图片(32px)图片(40px)CTA 的视觉节奏。

## Risks / Trade-offs

- **[图片加载失败显示空白]** → 设置 alt 文本 + 背景色占位，确保即使图片不加载页面结构也不崩
- **[图片体积影响加载速度]** → 建议图片控制在 200KB 以内，后续可通过工具压缩
- **[添加图片打破纯文字极简哲学]** → 图片尺寸受限于 600px，作为"补充"而非"主角"，文字仍是首屏核心
