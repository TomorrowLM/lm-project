## Context

从零构建手机产品官网首页。项目当前无任何代码文件，是一个全新的纯静态站点。目标是用最少的文件和最克制的视觉手段，建立一个有品牌感的线上入口。

约束：
- 纯 HTML + CSS，零 JavaScript 依赖
- 文字驱动视觉，不使用图片或插画
- 白底黑字，极简排版
- 文件结构：index.html + style.css 分离

## Goals / Non-Goals

**Goals:**
- 全屏 Hero 区在 3 秒内传达产品核心定位
- Footer 提供完整导航与法律信息
- 页面在桌面端和移动端均有良好阅读体验
- 纯 HTML/CSS 实现，零构建工具，可直接浏览器打开

**Non-Goals:**
- 不做 JavaScript 交互或动画
- 不做深色模式切换
- 不做导航栏（Nav 属于后续迭代）
- 不做 SEO 优化（后续迭代）
- 不做多页面路由

## Decisions

### 1. 文件结构：HTML + CSS 分离

**选择**: `index.html` + `style.css` 两个独立文件

**理由**: 分离关注点，HTML 负责语义结构，CSS 负责视觉表现。对于单页面站点，这比内联 `<style>` 更易维护，又比 CSS 模块化方案更简单。

**备选方案**:
- 单文件内联 CSS：更极简但不利于后续维护
- CSS Modules / 预处理器：过度工程化

### 2. 字体策略：系统字体栈

**选择**: 使用系统字体栈 `-apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, sans-serif`

**理由**: 零网络请求，即时渲染，各平台原生体验最佳。纯 HTML 项目引入外部字体（Google Fonts 等）会增加依赖和加载时间，与极简目标矛盾。

**备选方案**:
- Google Fonts 引入定制字体：增加网络请求，破坏离线可用性
- 仅用 `sans-serif`：过于粗糙，不同平台差异大

### 3. Hero 布局：Flexbox 垂直居中

**选择**: `min-height: 100vh` + Flexbox 垂直水平居中

**理由**: 最简洁的方式实现全屏居中效果，无需 JavaScript 计算视口高度，语义清晰。

### 4. 响应式策略：单一断点

**选择**: 仅在 `768px` 设置一个断点，区分桌面端与移动端

**理由**: 页面只有文字和链接，结构极简，不需要多断点精细调控。移动端字号适当缩小、间距收紧即可。

### 5. Footer 一行式布局

**选择**: 单行水平排列 `链接 · 社交 · 版权`，移动端换行堆叠

**理由**: 一行式与极简调性一致。传统四列 Footer 对只有两个链接分组的页面来说过重。

### 6. 颜色体系

**选择**:
- 背景：`#ffffff`（纯白）
- 主文字：`#111111`（近黑）
- 辅助文字：`#666666`（中灰）
- CTA 主按钮：`#111111` 填充 + 白字
- CTA 次按钮：透明底 + `#111111` 边框
- Footer 分隔线：`#e0e0e0`

**理由**: 黑白灰三级足够表达层次，无需品牌色。理性技术调性下，克制就是力量。

## Risks / Trade-offs

- **[文字驱动可能显得"空"]** → 通过字号对比（H1 64px vs 副标题 20px）和大量留白营造呼吸感，而非添加装饰
- **[无导航栏可能影响可发现性]** → Footer 承担全部导航功能，后续迭代可加 Nav
- **[系统字体在 Windows 上偏 Segoe UI，macOS 偏 San Francisco]** → 这是特性而非缺陷，各平台原生体验正是目标
- **[单文件无构建流程，后续扩展需手动维护]** → 接受此代价，当前规模不需要构建工具
